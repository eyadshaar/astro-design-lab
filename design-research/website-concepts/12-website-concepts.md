## Concept 1: The Canopy House — Boutique Hotel

**Industry:** Boutique Hotel / Hospitality
**Target audience:** Affluent travelers (35-55), experience-seekers, Pacific Northwest region
**Brand personality:** Grounded luxury — natural materials, local craft, unhurried pace

### Visual Identity
- **Palette:**
  - Background: #F8F5F0 (warm cream)
  - Text: #2D2A26 (warm charcoal)
  - Primary: #4A5D4A (forest green — the forest canopy)
  - Accent: #C8A97E (warm amber — lantern glow)
  - Dark: #1C1C1A (charcoal)
- **Typography:**
  - Display: Cormorant Garamond (elegant, editorial)
  - Body: DM Sans (clean, modern)
- **Metaphor:** "The Forest Journal" — the site feels like a naturalist's field notebook with pressed leaves, handwritten notes, and a sense of place

### Layout Strategy
- Magazine editorial layout — one dominant image per section
- Asymmetric grid — text and image interact in unexpected ways
- Generous whitespace — content breathes
- Navigation transparent over hero

### Hero Concept
Cinematic full-viewport image of the property in morning mist. Minimal text overlay: property name + "Book a room." Ghost button CTA. Dark vignette.

### Section List
1. Hero (cinematic property image)
2. Philosophy statement (one sentence about place)
3. Rooms: magazine-spread layout (not card grid)
4. Experience: curated activity list with editorial photos
5. Local Guide: itinerary-style recommendations
6. Gallery: masonry grid
7. Guest quote (large serif pull quote)
8. Final CTA (Reserve)
9. Footer with contact, address, social

### Motion
Slow Ken Burns on hero. Fade-in on scroll. No bounce.

### What makes it distinct
No stock photography of generic hotel rooms. Real property photography. Local guide as editorial content. Forest/natural motif carried through subtly.

### Design resources used
- Landing.love (cinematic hero pattern)
- Curated.design (editorial layout)
- Navbar Gallery (minimal transparent navbar)

## Concept 6: Stackflow — Developer Tool (CI/CD Platform)

**Industry:** Developer Tools / CI/CD
**Target audience:** DevOps engineers, platform engineers, engineering leads at tech companies
**Brand personality:** Precise, fast, no-fluff — the opposite of "supercharge your workflow"

### Visual Identity
- **Palette:**
  - Background: #0F1117 (near-black)
  - Surface: #1A1D27 (dark slate)
  - Text: #E6EDF3 (off-white)
  - Primary: #58A6FF (link blue)
  - Accent: #3FB950 (success green — pipelines, deploys)
  - Border: #30363D (subtle borders)
- **Typography:**
  - Display: Syne (geometric, distinctive)
  - Body: Inter (for this context, appropriate)
  - Mono: JetBrains Mono
- **Metaphor:** "The Pipeline" — the site is structured like a CI/CD pipeline. Sections are "stages." Content flows through in sequence.

### Layout Strategy
- Dark terminal aesthetic
- Monospace accents throughout
- Pipeline visualization as structural motif
- Clean status indicators and metrics

### Hero Concept
Terminal-style command-line interface. A deploy command that types out: `stackflow deploy --prod --now`. Below: "Deployed in 8 seconds." Dashboard showing deployment metrics.

### Section List
1. Hero (terminal + deploy output)
2. Pipeline visualization: how it works
3. Integrations: GitHub, GitLab, Bitbucket, Kubernetes
4. Metrics: deployment speed, success rate, time savings
5. Case studies: 3 specific engineering teams
6. Comparison: vs manual deploy vs Jenkins
7. Pricing (simple 3-tier)
8. Docs CTA
9. Footer

### Motion
Typewriter effect on terminal. Count-up on metrics. Tab switching on comparison.

### What makes it distinct
Developer tool sites all look the same. Stackflow uses pipeline visualization as the structural metaphor. Specific metrics, honest comparison, no buzzwords.

### Design resources used
- Component.gallery (command palette, terminal UI)
- Landing.love (terminal animation)
- Navbar.gallery (developer navbar)
